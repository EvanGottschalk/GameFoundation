Our code needs to have config files for the key values used in the games; all the logic should be in the code, but these JSON format config files should have the important values in key-value pairs.
Here are some examples:
1. controls.ts - this contains default information about the settings for controlling the character in the game
2. player.ts - this contains default information about the user's character's appearance, such as colors, sizes, file locations, etc. It should also include information about the character's movement speed, animations, abilities, items, sound effects, etc.
3. global.ts - this contains default information about the gameplay, generally, such as colors, sizes, file locations, etc. It should also include information how the physics apply to all things, default values for rewards, etc.
4. display.ts - this contains default information about how the game window itself appears on the screen
5. dialogue.ts - this contains default information about how chat messages appear on the screen within the game, how conversations with NPCs work, etc.
6. characters/CHARACTER_NAME.ts - this folder individual config files for each character. Each file contains default information about NPC characters' appearances, such as colors, sizes, file locations, etc. It should also include information about the characters' movement speed, animations, abilities, items, sound effects, NPC dialogue, etc.
7. zones/ZONE_NAME.ts - this folder individual config files for each different area in the game. Each file contains default information about the areas' appearances, such as colors, sizes, file locations, etc. It should also include information about the area, such as lists of items and objects with their coordinates, lists of NPCs with their coordinates, sound effects, music, etc. Coordinates are x and y values from 0 to 1. x: .5 means the bottom-left hand corner of the object is exactly halfway across the screen horizontally. x: 0 means the object is flush against the left side.
8. items/ITEM_NAME.ts - this folder individual config files for each different item in the game. Each file contains default information about the items' appearances, such as colors, sizes, file locations, etc. It should also include information about the item, such as their stats, sound effects, etc.
9. objects/OBJECT_NAME.ts - this folder individual config files for each different object in the game. Each file contains default information about the objects' appearances, such as colors, sizes, file locations, etc. It should also include information about the objects, such as their interactability, sound effects, etc.
